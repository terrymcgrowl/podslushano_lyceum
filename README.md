# Бот-предложка для канала "Подслушано"

## Что умеет

- Принимает от пользователей всё: текст, фото, видео, стикеры, gif, кружки, голосовые, документы
- Поддерживает медиагруппы (несколько фото/видео в одном сообщении)
- Пересылает в личку админу с кнопками "Опубликовать / Пропустить"
- Публикует в канал через copy_message (без плашки "переслано")
- Health check эндпоинт `/health` — нужен для cron-job чтобы Render free tier не засыпал

## Настройка

### 1. Создать бота

У @BotFather создать бота, получить токен.

### 2. Получить свой ADMIN_ID

Написать @userinfobot — он скажет ваш id.

### 3. Добавить бота в канал

Добавить бота администратором канала с правом публикации сообщений.

### 4. Получить CHANNEL_ID

Вариант А — если канал публичный: просто `@username` канала.  
Вариант Б — если приватный: переслать любое сообщение из канала боту @JsonDumpBot, взять значение `forward_from_chat.id`.

### 5. Деплой на Render

1. Залить код в GitHub репозиторий
2. Зайти на render.com, создать Web Service, подключить репо
3. В Environment Variables добавить:
   - `BOT_TOKEN` — токен бота
   - `ADMIN_ID` — ваш telegram id (число)
   - `CHANNEL_ID` — id или @username канала
4. Build Command: `pip install -r requirements.txt`
5. Start Command: `python bot.py`

### 6. Настроить cron-job чтобы Render не засыпал

На cron-job.org (бесплатно) добавить задачу:
- URL: `https://ваш-сервис.onrender.com/health`
- Schedule: `*/10 * * * *` (каждые 10 минут)

## Важные детали

**Медиагруппы.** Telegram присылает фото/видео из одного альбома отдельными апдейтами.
Бот собирает их в течение 1 секунды и пересылает пачкой.

**Публикация.** Используется `copy_message` — сообщение публикуется без плашки "переслано от пользователя".
Для медиагрупп кнопка хранит `message_id` первого сообщения из группы и `user_id` отправителя.
При нажатии "Опубликовать" копируется только первое сообщение медиагруппы — если нужно копировать всю группу,
нужна БД для хранения всех message_id группы (см. примечание ниже).

**Примечание про медиагруппы при публикации.** Telegram Bot API не умеет копировать медиагруппу целиком
одним вызовом. Чтобы публиковать полные альбомы, нужно хранить все message_id группы.
Простейший бесплатный вариант — добавить SQLite (файл на диске, но Render free tier не имеет
персистентного диска) или использовать Supabase/PlanetScale free tier.
Альтернатива — хранить в памяти (теряется при рестарте, для MVP приемлемо).
